              Описание задания
Нужно реализовать сервис интеграции для huntflow.
Сервис должен:
* при переводе кандидата по вакансии на этап hired назначать ему тег нанят (если такого тега
нет, то добавить)
* при открытии вакансии: искать кандидатов (через API метод поиска кандидатов) по названию должности (поле position) в данных вакансии (искать только тех, кто не был добавлен ни на одну из вакансий); первого найденного кандидата назначать на созданную вакансию на первый статус вакансии. Кандадатов искать тоже по названию должности (поле position). Если кандидатов не найдено, то ничего не делать.
Пояснения:
   * Этап hired - этап движения кандидата по вакансии, см. https://api.huntflow.ru/v2/docs#get-/accounts/-account_id-/vacancies/statuses
 нужно из всех этапов выбрать один, у которого type == "hired"
   * теги кандидата - см. https://api.huntflow.ru/v2/docs#get-/accounts/-account_id-/tags
 https://api.huntflow.ru/v2/docs#post-/accounts/-account_id-/tags
   * добавить вакансию в базу можно через веб-интерфейс хантфлоу, после логина нужно нажать на плюс в левом нижнем углу и выбрать в меню "Добавить вакансию"
   * аналогично с добавлением кандидатов - нажать на плюс и выбрать "Добавить кандидата"
   * статусы вакансии можно получить через метод API https://api.huntflow.ru/v2/docs#get-/accounts/-account_id-/vacancies/statuses
   * открытие вакансии - когда вакансия получает статус “OPEN“ (в вебхуке на вакансию придет соответствующий vacancy_log)

Входные данные:
Для выполнения тестового задания предоставляется песочница https://dev-100.huntflow.dev
нужно передать рекрутеру Хантфлоу адрес электронной почты который будет являться логином для входа в интерфейс Хантфлоу.
Рекрутер, в свою очередь передаст адрес песочницы, пароль для входа.
Песочница представляет собой полноценный аккаунт в приложении Хантфлоу, со своей собственной
организацией. Доступ выдается на 14 дней.
Адрес API сервера: https://dev-100-api.huntflow.dev
Документация API доступна там же: https://dev-100-api.huntflow.dev/v2/docs
Для выполнения тестового задания потребуется реализовать http-сервис. Сервис должен быть доступен с хоста на котором работает тестовая песочница huntflow. Чтобы это сделать достаточно поднять сервис локально на своем компьютере и воспользоваться ngrok (https://ngrok.com) или альтернативами, например, Localtunnel ~ Expose yourself to the world
Также нужно настроить вебхуки в интерфейсе Хантфлоу, это можно сделать кликнув по названию
организации в правом верхнем углу и выбрав "настройки" в выпадающем меню. Далее перейти к разделу "API" (нижний пункт в списке слева на странице настроек).
Документация по вебхукам: https://github.com/huntflow/api/blob/master/ru/webhooks.md  (см. версию 2.0)
Для работы с API нужно сгенерировать токен, он генерируется на этой же странице.
В целом схема работы реализованной интеграции должна получиться такой:
      * регистрируете вебхук для изменений по кандидатам
      * регистрируете вебхук для изменений по вакансиям
      * реализуете соответствующие эндпоинты в своем сервисе
      * при движении кандидата Хантфлоу пошлет запрос на зарегистрированный вебхук по кандидатам
      * в вашем сервисе вы обрабатываете запрос, в процессе обработки  делаете запросы в API huntflow используя API-токен (генерируется там же где регистрируются вебхуки)
Для реализации можно взять готовый шаблон приложения, в котором есть заготовки для нужных вебхуков: https://github.com/huntflow/huntflow-integration-sample-template - Подключить для просмотра
Либо реализовать сервис самостоятельно, если есть время/желание (для оценки тестового в целом не принципиально какой из способов вы выберите).
Если будете реализовывать на базе указанного шаблона, то сделайте его форк а свои изменения оформите в виде pull request к вашему форку.
